# 디바이스 예약 관리 화면

디바이스 예약을 타임라인 기반으로 관리할 수 있는 프론트엔드 과제입니다.
실제 사용 시의 흐름과 UX를 고려하여 예약 생성, 조회, 히스토리 관리 기능을 구현했습니다.

---

## 1. 프로젝트 실행 방법

```bash
npm install
npm start # vite + json-server 실행
```

- Vite 기반 개발 서버와 json-server가 함께 실행됩니다.

## 2. 테스트 데이터 생성 방법

```bash
node create-reservations.js > db.json
```

- json-server에서 사용할 예약/디바이스 테스트 데이터를 생성합니다.

- 날짜 기준 ± 일정 범위의 예약 데이터가 자동 생성됩니다.

## 3. 사용 기술 스택

- React 18 + TypeScript
- Vite
- MUI v7
- React Router
- React Query
- React Hook Form + Zod
- Zustand
- json-server

## 4. 구현 기능

### 예약 타임라인

- 디바이스별 예약 현황을 시간 축 기반으로 시각화
- 현재 시간 표시 및 지나간 시간 영역 비활성화
- 마우스 인터랙션 기반 예약 생성 버튼 노출

### 예약 관리

- 예약 생성 / 취소
- 예약 상세 모달 (본인 예약 여부에 따른 버튼 분기)

### 히스토리

- 예약 히스토리 테이블
- 필터 기능 (OS, 디바이스, 사용자, 기간, 상태)
- 페이지네이션 처리

## 5. 고민했던 부분

- 타임라인 UI에서 시간, 상태, 인터랙션을 어떻게 직관적으로 표현할지
- 예약 상태에 따라 가능한 동작을 명확하게 구분하는 UX의 구현 방법
- 필터 적용/취소/초기화 흐름을 실제 사용 관점에서 자연스럽게 설계하는 부분 등

여러 화면을 설계하며 어떤 부분을 재활용하고 어떻게 구현해야 하는지 고민하며 작업했습니다.

특히나 이 화면의 핵심인 타임라인을 어디까지 얼마나 어떻게 구현해야 하는지 많이 고민했습니다.

## 6. 아쉬운 점
